# 功过豆小程序设计文档

- 日期：2026-04-27
- 阶段：设计确认稿
- 适用范围：第一版 MVP 及后续二、三阶段演进

## 1. 项目概述

本项目是一个基于微信小程序的个人修身记录工具。用户通过非常轻量的交互，在每日生活中记录“功”与“过”，以赤豆与青豆作为具象化表达，并在每天结束时获得当日小结。

第一版的产品目标不是做成社交竞争产品，也不是严肃说教工具，而是做成一个用户愿意每天打开 10 秒钟、轻轻记一笔、再看一眼今日结果的“轻仪式感修身小程序”。

产品的核心体验为：

1. 打开首页即可记录，不需要学习复杂规则。
2. 每次记录都获得即时、直观、有手感的视觉反馈。
3. 每日结束时有一个完整、克制、有余味的结算体验。
4. 支持分享小程序本身与当日结果，但不以社交曝光为核心。

## 2. 产品定位

### 2.1 产品定位

产品定位为“每日功过自记器”，强调用户对自己一天言行的轻量观照，而不是外部评价。

关键词：

- 自我记录
- 轻仪式感
- 东方器物感
- 日结收束
- 温和自省

### 2.2 非目标

第一版明确不追求以下方向：

1. 不做交易、支付或商业闭环。
2. 不做好友监督、公开互评或排行榜。
3. 不做复杂日记、长篇内容创作。
4. 不做强惩罚机制或羞辱式提醒。
5. 不做宗教化、命理化或过重说教表达。

## 3. 用户与使用场景

### 3.1 核心用户

核心用户是愿意进行轻量自我反思、偏好有趣但不浮夸表达方式的个人用户。

用户特征：

1. 愿意尝试简单新鲜的小工具。
2. 不追求复杂功能，更看重每天是否容易坚持。
3. 希望通过一个轻量机制提醒自己，不需要被强评价。

### 3.2 核心使用场景

1. 白天某个时刻，用户做了一件自己认为值得记功的事，打开小程序点一下“记一功”。
2. 白天某个时刻，用户觉得自己有一个需要提醒自己的过失，打开小程序点一下“记一过”。
3. 晚上或第二天回看时，查看当日功过结果和近几日变化。
4. 在觉得当天结果有意思或有仪式感时，分享当日小结给朋友，或分享小程序本身邀请他人使用。

## 4. 核心玩法规则

### 4.1 槽位与豆子

首页包含两个主要槽位：

1. 功槽：用于承载赤豆，表示当日“功”。
2. 过槽：用于承载青豆，表示当日“过”。

### 4.2 记录规则

用户可以执行两种动作：

1. 记一功
2. 记一过

### 4.3 抵消规则

功与过互相抵消，规则写死如下：

1. 当用户执行“记一功”时：
   - 若过槽内存在至少一颗青豆，则优先抵消一颗青豆。
   - 若过槽为空，则向功槽新增一颗赤豆。
2. 当用户执行“记一过”时：
   - 若功槽内存在至少一颗赤豆，则优先抵消一颗赤豆。
   - 若功槽为空，则向过槽新增一颗青豆。

### 4.4 每日结算规则

1. 每天独立结算，不跨天抵消。
2. 以中国时区自然日为边界进行切日。
3. 每日结果仅归类为三种状态：
   - 偏功
   - 相抵
   - 偏过
4. 第二天开始时，前一天进入历史，不把剩余豆子带入新的一天。
5. 若用户未在零点时停留在小程序内，则在下一次打开小程序时自动完成前一日结算。

### 4.5 记录内容策略

第一版采用“极简主路径 + 可选补充”的记录方式：

1. 用户默认只需点击一次即可完成记录。
2. 记录后可选补一句简短缘由。
3. 第一版不强制分类，不要求用户每次写内容。

该策略用于同时满足低门槛记录与后续日结回味。

## 5. 第一版 MVP 范围

第一版只实现以下 5 项核心能力：

1. 今日功过案首页
2. 即时加豆与抵消反馈
3. 每日结算页
4. 历史查看页
5. 分享能力

### 5.1 今日功过案首页

首页为默认进入页，承担最关键的使用动作。用户打开小程序后应立即理解玩法，并能在极短时间内完成一次记录。

首页元素：

1. 日期
2. 功槽
3. 过槽
4. 今日净结果提示
5. “记一功”按钮
6. “记一过”按钮
7. 通往日结页与历史页的轻量入口

### 5.2 即时反馈

每次记录必须立即反馈：

1. 豆子弹出并落入槽中。
2. 若发生抵消，展示抵消动画。
3. 页面出现一条极短反馈短句。

### 5.3 每日结算页

日结页用于收束一天，不进行说教，只给出结果与一句短结语。

页面内容：

1. 日期
2. 今日赤豆数
3. 今日青豆数
4. 抵消后净结果
5. 状态标签：偏功 / 相抵 / 偏过
6. 一句结语
7. 分享按钮

### 5.4 历史查看页

第一版历史页保持克制，默认仅提供近 7 天的按日记录查看。

每条历史至少展示：

1. 日期
2. 当日状态
3. 功豆数量
4. 过豆数量

用户点击某日后，可查看当天简要内容和可选备注。

### 5.5 分享能力

第一版支持两类分享：

1. 分享小程序本身
2. 分享当日结算结果卡片

分享原则：

1. 默认不暴露用户具体记录内容。
2. 默认只分享结果感与产品气质。
3. 重点是邀请与表达，不是公开评判。
4. 第一版默认不把用户填写的可选备注带入分享卡片。

## 6. 页面与交互设计

### 6.1 页面结构

第一版维持三页主结构：

1. 今日页
2. 日结页
3. 历史页

### 6.2 今日页交互

用户路径：

1. 打开小程序
2. 看到今日功过案
3. 点击“记一功”或“记一过”
4. 观看落豆或抵消反馈
5. 继续记录，或稍后查看日结

交互原则：

1. 主动作应在一屏内完成。
2. 不引导用户先学习复杂规则。
3. 不弹出重表单打断节奏。

### 6.3 日结页交互

用户在日结页的主要行为：

1. 查看今日结果
2. 体会一句结语
3. 选择是否分享

日结页应该有“收一页”的感觉，而不是统计报表。

### 6.4 历史页交互

历史页应强调“积累感”，而不是复杂分析。

用户应快速感受到：

1. 自己是否持续记录了几天
2. 近几天整体偏向如何
3. 某一天是否值得回看

## 7. 视觉与美术方向

### 7.1 总体视觉方向

推荐视觉方向为“轻趣东方器物感”。

定义：

1. 有东方感，但避免老气仿古。
2. 有趣味，但避免幼态卡通。
3. 有质感，但避免冷淡空洞。

### 7.2 首页视觉意象

首页像一张可把玩的“小案台”，而不是常规打卡页面。

核心视觉元素：

1. 米纸感背景
2. 温润木案或木质承托结构
3. 两个器物感槽位
4. 圆润有手感的豆子

### 7.3 色彩策略

主色建议：

1. 背景：米白、宣纸白
2. 基底：浅木色、温润棕
3. 功豆：豆赤、朱砂偏暖红
4. 过豆：青黛、青灰偏冷色
5. 点缀：墨灰

色彩目标是安静、温暖、清楚，不走刺眼高饱和路线。

### 7.4 动效策略

动效是吸引力关键之一，应聚焦以下瞬间：

1. 豆子出现
2. 豆子落槽
3. 功过抵消
4. 日结收束

动效目标：

1. 短
2. 清楚
3. 有手感
4. 不拖慢操作节奏

## 8. 文案与气质方向

### 8.1 整体语气

文案语气定义为：

1. 温和
2. 克制
3. 略有古意
4. 不说教

### 8.2 文案禁区

避免以下表达风格：

1. 训诫式
2. 鸡汤式
3. 玄学式
4. 羞辱式

### 8.3 即时反馈示例

示例方向：

1. 善念记下了
2. 此过已知
3. 功过相抵
4. 今日仍可再正

### 8.4 日结结语方向

偏功、相抵、偏过三类结语应体现以下原则：

1. 只总结当天状态，不定义用户人格。
2. 留有余地，不制造负担。
3. 读起来像轻轻收束，而非高声评判。

具体文案后续细化打磨。

## 9. 数据结构建议

第一版建议采用以下核心数据对象：

1. 用户
2. 当日记录
3. 单次动作记录
4. 日结结果

### 9.1 用户

用户只保留第一版必要信息，用于识别和持有个人记录。

### 9.2 当日记录

字段建议：

1. 日期
2. 当日赤豆数
3. 当日青豆数
4. 净结果
5. 是否已结算
6. 可选备注列表

### 9.3 单次动作记录

字段建议：

1. 时间
2. 类型：功 / 过
3. 是否发生抵消
4. 可选短备注

### 9.4 日结结果

字段建议：

1. 日期
2. 功数
3. 过数
4. 净值
5. 状态：偏功 / 相抵 / 偏过
6. 结语
7. 是否分享过

## 10. 合规与发布边界

基于当前讨论，项目按个人主体小程序路线规划。

第一版边界：

1. 不涉及交易与支付。
2. 不以品牌经营为主要目标。
3. 主轴为个人记录修身工具。
4. 服务类目优先按工具类方向评估，不走游戏类。

上线前需重点处理：

1. 个人主体注册
2. 服务类目确认
3. 用户隐私保护指引
4. 备案要求

## 11. 阶段演进规划

### 11.1 第一阶段

目标：验证“个人记录是否成立”

核心特征：

1. 自己记
2. 自己看
3. 自己分享

### 11.2 第二阶段

目标：引入轻监督关系，提升坚持率

候选能力：

1. 邀请朋友围观结果摘要
2. 修身同伴关系
3. 是否完成当日记录的轻提醒
4. 自愿公开一句当日小结

### 11.3 第三阶段

目标：在不破坏修身气质的前提下引入适度社交比较

候选能力：

1. 连续记录天数比较
2. 轻量榜单
3. 七日主题挑战

原则：

1. 先自记，再互守，再彼此看见。
2. 不把产品做成道德排行榜。

## 12. 成功判断标准

第一版是否值得继续推进，重点不看大而空的注册量，而看以下信号：

1. 用户是否愿意连续记录 3 天以上
2. 用户是否愿意看完日结页
3. 用户是否愿意分享小程序或当日卡片
4. 用户是否产生“想拉朋友一起坚持”的自然意愿

只要前两项成立，第二阶段就有推进价值。

## 13. 本稿结论

本设计稿确认第一版采用以下总路线：

1. 产品定位：轻仪式感的每日修身记录工具
2. 交互主轴：记一功 / 记一过 / 抵消 / 日结
3. 气质方向：外在轻松有趣，内核克制自省
4. 视觉方向：轻趣东方器物感
5. 上线策略：先做个人记录成立，再做朋友监督，再考虑轻社交比较

该方案适合进入后续实现计划阶段。
