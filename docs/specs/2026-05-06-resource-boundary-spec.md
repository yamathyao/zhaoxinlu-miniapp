# 默认资源边界与可替换静态资源规范

- 日期：2026-05-06
- 状态：设计确认稿
- 范围：默认主题资源边界与未来静态资源分离预留

## 1. 目标

当前版本必须保持默认资源完整内置、离线可用，并且让页面和组件只依赖配置入口读取话术、主题与资源路径。后续若引入外部皮肤包，只允许替换静态资源和话术，不允许替换页面逻辑、路由或数据模型。

## 2. 当前边界

默认版本的实现边界如下：

- 页面和组件通过 `services/copy-service.js`、`services/theme-service.js`、`services/resource-provider.js` 读取可变内容。
- 话术统一收敛在 `miniprogram/config/default-copy.js`。
- 主题 token、主题名与资源映射统一收敛在 `miniprogram/config/default-theme.js`。
- 资源格式约束收敛在 `miniprogram/config/resource-schema.js`。
- 默认资源必须在无网络时继续可用。

## 3. 可抽离的资源

适合抽取为静态资源文件的内容包括：

- 首页背景纸纹和底纹
- 匣盖、匣身、木纹、封条、印章等视觉层
- 纸符纹理、卡片底图和心灯图案
- 以后可能替换的图标、徽记、小装饰件

这些内容应放在 `miniprogram/assets/themes/<theme-id>/` 下，由主题映射决定路径。默认主题目前使用 `heartLamp` 资源槽描述心灯图片；若资源缺失，组件保留 CSS 心灯兜底。

## 4. 不可抽离的内容

以下内容不得放入外部资源包，也不得由外部资源直接覆盖：

- JS 行为逻辑和事件处理
- WXML 结构和页面路由
- WXSS 布局骨架、动画关键帧和交互状态
- 本地存储模型与归档规则
- 页面进入、封存、解封、查看符文等业务流程

## 5. 推荐目录

```text
miniprogram/
  assets/
    themes/
      default/
        bg-paper.png
        box-lid.png
        box-body.png
        paper-slip.png
        seal.png
        heart-lamp.png
        card-bg.png
  config/
    default-copy.js
    default-theme.js
    resource-schema.js
  services/
    copy-service.js
    theme-service.js
    resource-provider.js
```

## 6. 迁移顺序

建议按以下顺序逐步替换：

1. 首页背景纸纹
2. 功匣 / 过匣主体纹理
3. 纸符纹理
4. 封存卡片底图
5. 心灯、印章与封条

迁移时应保持默认主题始终存在，任何可选主题都必须回退到默认主题。

## 7. 验证方式

- 默认资源在无网络条件下仍能打开首页、写入、封存和查看归档。
- 页面与组件不再直接写死主要话术和主题路径。
- 资源路径缺失时自动回退默认主题。
- 外部资源包仅能覆盖 `copy`、`theme`、`assets`。

## 8. 后续约束

未来若引入 GitHub 或 CDN 资源下载，必须先满足：

- 域名合法且可稳定访问
- 资源包做版本与字段校验
- 不执行远程 JS
- 不让资源包修改页面逻辑或存储结构
